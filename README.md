# gisila_studio

> Web-based admin interface for gisila models, inspired by Django admin.
> Auto-generates CRUD UI from `TableMeta` registrations and serves it
> over Shelf — zero frontend tooling required.

```dart
import 'package:gisila_studio/gisila_studio.dart';

final studio = GisilaStudio(db: db, title: 'Blog Admin');

studio.register<User>(
  UserTable.metadata,
  displayName: 'User',
  listDisplay: ['id', 'first_name', 'email', 'date_joined'],
  searchFields: ['first_name', 'last_name', 'email'],
  readonlyFields: ['id', 'date_joined'],
  ordering: ['-date_joined'],
);

studio.register<Post>(
  PostTable.metadata,
  listDisplay: ['id', 'title', 'author_id', 'published_at'],
  searchFields: ['title'],
  ordering: ['-published_at'],
);

// Mount under /studio on an existing router:
router.mount('/studio', studio.handler(prefix: '/studio'));
```

Open `http://localhost:8080/studio` and you get a fully functional
list / add / change / delete interface for every registered model —
no code generation, no JavaScript build step.

---

## Install

```yaml
# pubspec.yaml
dependencies:
  gisila_studio:
    path: ../gisila_studio   # replace with path / git / pub coordinate
  gisila_orm:
    path: ../gisila_orm
```

`gisila_orm` is a peer dependency: it must be present so that the
`TableMeta<T>` generated by `build_runner` is available at runtime.

## Quick start

### 1. Generate your models

Follow the [gisila_orm setup](../gisila_orm/README.md) to describe your
tables in a `.gisila.yaml` file and run `build_runner`. This produces
`FooTable.metadata` for each model.

### 2. Create the studio

```dart
import 'package:gisila_studio/gisila_studio.dart';
import 'package:shelf_router/shelf_router.dart';
import 'package:shelf/shelf_io.dart' as io;

// (db is a Database instance from gisila_orm)
final studio = GisilaStudio(db: db, title: 'My App Admin');
```

### 3. Register models

```dart
studio.register<User>(
  UserTable.metadata,
  displayName: 'User',
  displayNamePlural: 'Users',
  listDisplay: ['id', 'email', 'is_active', 'date_joined'],
  searchFields: ['email'],
  readonlyFields: ['id', 'date_joined'],
  ordering: ['-date_joined'],
  group: 'Accounts',
);
```

### 4. Mount and serve

```dart
final router = Router();
router.mount('/admin', studio.handler(prefix: '/admin'));

final server = await io.serve(router.call, 'localhost', 8080);
print('Admin at http://localhost:${server.port}/admin');
```

Serve standalone (without a parent router):

```dart
await io.serve(studio.handler(prefix: ''), 'localhost', 8081);
```

## Authentication

If `STUDIO_USERNAME` and `STUDIO_PASSWORD` environment variables are
set, every page is gated behind a login form. Credentials can also be
passed directly to the constructor:

```dart
final studio = GisilaStudio(
  db: db,
  title: 'My App Admin',
  username: 'admin',
  password: 'supersecret',
);
```

Sessions are managed in-process with `HttpOnly; SameSite=Lax` cookies.
A logout link appears in the sidebar header when authentication is
enabled.

If neither the environment variables nor the constructor arguments are
provided the studio runs without authentication — suitable for local
development only.

## `GisilaStudio` API

```dart
GisilaStudio({
  required Database db,
  String title = 'Gisila Studio',
  String? username,   // falls back to STUDIO_USERNAME env var
  String? password,   // falls back to STUDIO_PASSWORD env var
})
```

### `register<T>`

```dart
void register<T>(
  TableMeta<T> meta, {
  String? displayName,        // human-readable singular; default: humanize(tableName)
  String? displayNamePlural,  // default: displayName + 's'
  List<String>? listDisplay,  // columns in list view; default: first 6
  List<String> searchFields,  // ILIKE search targets
  List<String> readonlyFields,// shown on forms but not editable
  List<String> excludeFields, // hidden from forms entirely
  int pageSize = 25,
  List<String> ordering,      // default sort; prefix with '-' for DESC
  String? group,              // sidebar section label
})
```

### `handler`

```dart
Handler handler({String prefix = '/studio'})
```

Returns a Shelf `Handler` suitable for `router.mount(prefix, ...)`.
Pass the same string as `prefix` so all internal links resolve
correctly. Pass an empty string when serving standalone.

## Features

| Feature | Details |
| ------- | ------- |
| List view | Paginated table with configurable columns, search, and sort. |
| Add / change forms | Auto-generated from column metadata; respects `readonlyFields` and `excludeFields`. |
| Delete | Single-row delete from the change form or the list action bar. |
| Search | `ILIKE '%query%'` across all `searchFields`. |
| Ordering | Click column headers in the list; default order via `ordering`. |
| Sidebar groups | `group:` parameter groups models under a labelled section. |
| Authentication | Session-based login form; credentials from env vars or constructor. |
| Custom title | Set `title` to brand the admin interface. |

---

## Pages served

| Path | Description |
| ---- | ----------- |
| `{prefix}/` | Dashboard — lists all registered models. |
| `{prefix}/{model}/` | List view — paginated rows with search and sort. |
| `{prefix}/{model}/add/` | Add form. |
| `{prefix}/{model}/{pk}/change/` | Change (edit) form. |
| `{prefix}/{model}/{pk}/delete/` | Delete confirmation. |
| `{prefix}/login/` | Login form (only when auth is enabled). |
| `{prefix}/logout/` | Clears session cookie and redirects to login. |
